¿Qué es EtherChannel?

Concepto

Definición y propósito

EtherChannel (también llamado Link Aggregation o Port Aggregation) agrupa entre 2 y 8 puertos físicos en una sola interfaz lógica llamada Port-Channel.

El switch vecino lo ve como un único enlace. STP también lo trata como un solo puerto, evitando que bloquee los enlaces redundantes.

SW-A SW-B Gi0/1 ══════════════════ Gi0/1 Gi0/2 ══════════════════ Gi0/2 → Lógicamente: Gi0/3 ══════════════════ Gi0/3 Gi0/4 ══════════════════ Gi0/4 └── Port-Channel 1 (4 Gbps) ──┘

Beneficios clave

Ancho de banda
Suma todos los enlaces
Redundancia
Si cae un link, sigue
STP
Lo ve como 1 puerto
Config. única
Se configura en Port-Channel

Requisitos para formar EtherChannel

Todos los puertos físicos agrupados deben tener exactamente los mismos parámetros:

ParámetroDebe ser igual en todos los puertos
VelocidadEj: todos en 1000 Mbps
DúplexTodos full-duplex
Modo switchportTodos access o todos trunk
VLAN permitidas (si trunk)Mismas VLANs permitidas
VLAN nativa (si trunk)Misma VLAN nativa
VLAN access (si access)Misma VLAN de acceso

Configuración PAgP — Desirable / Auto

Se configuran los puertos físicos y se asignan al Port-Channel con channel-group.

Cisco IOS — PAgP 
# En SW-A (modo Desirable — inicia negociación)
en
conf t
int range gig0/1 - 4
channel-group 1 mode desirable
exit
!
# En SW-B (modo Auto — espera)
int range gig0/1 - 4
channel-group 1 mode auto
exit

IEEE 802.1AX

Características LACP

Estándar original
IEEE 802.3ad
Estándar actual (moderno)
IEEE 802.1AX
Modos
Active / Passive / On
Interoperabilidad
Cualquier fabricante
Links activos máx.
8 activos
Links standby máx.
8 en reserva

Si falla un enlace activo, un enlace de reserva (standby) se activa automáticamente. Esto lo diferencia de PAgP, que no tiene standby.

Configuración LACP — Paso a paso

Tres pasos para crear un EtherChannel LACP con trunk.

Paso 1 — Seleccionar las interfaces con interface range

Cisco IOS — Paso 1 
S1(config)# interface range FastEthernet 0/1 - 2

Paso 2 — Crear el Port-Channel con channel-group y modo LACP

Cisco IOS — Paso 2 
S1(config-if-range)# channel-group 1 mode active
Creating a port-channel interface Port-channel 1
S1(config-if-range)# exit

Paso 3 — Configurar el Port-Channel como trunk

Cisco IOS — Paso 3 
S1(config)# interface port-channel 1
S1(config-if)# switchport mode trunk
S1(config-if)# switchport trunk allowed vlan 1,2,20

La configuración de trunk y VLANs se aplica sobre el port-channel, no sobre las interfaces físicas individuales. Lo que se configura en el Port-Channel se aplica automáticamente a todos los puertos físicos del grupo.

Versión completa en S1 (Active) y S2 (Passive):

Cisco IOS — Ambos switches 
# S1 — modo Active (inicia negociación)
interface range FastEthernet 0/1 - 2
channel-group 1 mode active
exit
interface port-channel 1
switchport mode trunk
switchport trunk allowed vlan 1,2,20
!
# S2 — modo Passive (responde, no inicia)
interface range FastEthernet 0/1 - 2
channel-group 1 mode passive
exit
interface port-channel 1
switchport mode trunk
switchport trunk allowed vlan 1,2,20
🟡

Static EtherChannel (modo On)

Sin negociación

Modo On — Sin protocolo de negociación

Fuerza el EtherChannel sin PAgP ni LACP. Ambos lados deben estar en On. No hay mensajes de control — si hay un problema de configuración, no se detecta automáticamente.

Cisco IOS — Static On 
# Ambos switches deben estar en On
int range gig0/1 - 4
channel-group 1 mode on
exit
