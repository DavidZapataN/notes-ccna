Comandos Básicos

Nivel 1

Nombre del dispositivo (hostname)

Lo primero que se configura. Identifica el dispositivo en la red.

Cisco IOS 
# Entrar a modo privilegiado y luego configuración global
en
conf t
hostname SW-PRINCIPAL

Contraseñas y seguridad básica

Proteger el acceso al dispositivo con contraseñas cifradas.

Cisco IOS 
# Contraseña modo privilegiado (cifrada)
enable secret MiClave123
# Cifrar todas las contraseñas en texto plano
service password-encryption
# Contraseña de consola
line console 0
password cisco
login
exit

Banner y utilidades

Mensaje de advertencia al conectarse y deshabilitar búsqueda DNS innecesaria.

Cisco IOS 
# Mensaje de bienvenida/advertencia
banner motd #Acceso solo personal autorizado#
# Evitar que el router busque DNS al escribir mal un comando
no ip domain-lookup
# Guardar configuración
do write
# o también:
do copy run start

Acceso remoto por Telnet / SSH

Configurar líneas VTY para acceso remoto al dispositivo.

Cisco IOS 
line vty 0 4
password cisco
login
transport input telnet ssh
exit

Interfaces

Nivel 2

Asignar IP a una interfaz (Router)

En routers, las interfaces físicas reciben IP directamente. Siempre activar con no shutdown.

Cisco IOS 
en
conf t
int gig0/0/0
description Enlace a Switch Principal
ip address 192.168.1.1 255.255.255.0
no shutdown
exit

Ver estado de interfaces

Cisco IOS — Verificación 
# Resumen de todas las interfaces
do show ip int brief
# Detalle de una interfaz específica
do show int gig0/0/0

Default Gateway

Nivel 3

Default gateway para switches de capa 2

Un switch L2 no enruta, pero necesita un gateway para poder ser administrado remotamente desde otra red.

Cisco IOS 
en
conf t
ip default-gateway 10.40.0.100

VLANs

Nivel 4

Crear VLANs y asignar nombres

Las VLANs segmentan la red lógicamente. Cada VLAN es un dominio de broadcast independiente.

Cisco IOS 
en
conf t
!
vlan 10
name ESTUDIANTE
!
vlan 20
name PROFESOR
!
vlan 30
name ADMINISTRATIVO
!
vlan 40
name MANAGE
!
vlan 50
name NATIVA
exit
!
do show vlan brief

Puertos Trunk / Access

Nivel 5

Puertos Trunk (Gig1/0/1 – 4)

Los puertos trunk transportan tráfico de múltiples VLANs. Se usan entre switches o entre switch y router.

Cisco IOS 
en
conf t
int range gig1/0/1 - 4
switchport mode trunk
switchport trunk native vlan 50
exit
!
do show interfaces trunk

Puertos Access por VLAN

Los puertos access pertenecen a una sola VLAN. Se conectan a dispositivos finales (PCs, impresoras, etc.).

Cisco IOS 
# VLAN 10 - ESTUDIANTE (puertos 5-6)
int range gig1/0/5 - 6
switchport mode access
switchport access vlan 10
exit
!
# VLAN 20 - PROFESOR (puertos 7-10)
int range gig1/0/7 - 10
switchport mode access
switchport access vlan 20
exit
!
# VLAN 30 - ADMINISTRATIVO (puertos 11-13)
int range gig1/0/11 - 13
switchport mode access
switchport access vlan 30
exit
!
# VLAN 40 - MANAGE (puertos 14-15)
int range gig1/0/14 - 15
switchport mode access
switchport access vlan 40
exit
!
# VLAN 50 - NATIVA (puertos 16-19)
int range gig1/0/16 - 19
switchport mode access
switchport access vlan 50
exit
!
do show vlan brief

IP del Switch (SVI)

Nivel 6

Switch Virtual Interface (SVI)

En switches L3, se asigna IP a interfaces VLAN virtuales. La VLAN 40 (MANAGE) es la de gestión del switch.

Cisco IOS 
en
conf t
# IP de gestión del switch (VLAN 40)
int vlan 40
ip address 10.40.0.6 255.255.0.0
no shutdown
!
# Default gateways (inter-VLAN) — una IP por VLAN
int vlan 10
ip address 10.10.0.100 255.255.0.0
no shutdown
!
int vlan 20
ip address 10.20.0.100 255.255.0.0
no shutdown
!
int vlan 30
ip address 10.30.0.100 255.255.0.0
no shutdown
!
int vlan 40
ip address 10.40.0.100 255.255.0.0
no shutdown
!
int vlan 50
ip address 10.50.0.100 255.255.0.0
no shutdown

Routing (Inter-VLAN)

Nivel 7

Habilitar routing en el switch L3

Con ip routing el switch puede enrutar tráfico entre VLANs sin necesidad de un router externo.

Cisco IOS 
en
conf t
ip routing
!
# Verificar rutas
do show ip route

IPv6 / SLAAC / NDP

Nivel 8

SLAAC — Stateless Address Autoconfiguration

Permite que un dispositivo configure su propia dirección IPv6 automáticamente usando el prefijo anunciado por el router (RA) y su dirección MAC (EUI-64).

NDP — Neighbor Discovery Protocol

Reemplaza a ARP en IPv6. Usa mensajes ICMPv6 para descubrir vecinos, prefijos y routers.

🏓 Conectividad (Ping)
Echo Request128
Echo Reply129
🔗 Vecino L2 (reemplaza ARP)
Neighbor Solicitation (NS)135
Neighbor Advertisement (NA)136
🌍 Vecino L3 (prefijo)
Router Solicitation (RS)133
Router Advertisement (RA)134
📡 Direcciones Multicast
RS destino (All Routers)FF02::2
RA destino (All Nodes)FF02::1

Configurar IPv6 en una interfaz

Cisco IOS 
en
conf t
int gig0/0/0
# Dirección link-local (solo en la red local)
ipv6 address fe80::1 link-local
# Dirección global unicast
ipv6 address 3000::1/64
no shutdown
!
# Ver IPv6 en la interfaz
do show ipv6 interface gig0/0/0

SLAAC en un switch (obtener IPv6 del router)

El switch solicita su dirección IPv6 automáticamente al router mediante RS/RA.

Cisco IOS 
# Reiniciar el dispositivo primero
reload
# Luego en la interfaz:
ipv6 address autoconfig
